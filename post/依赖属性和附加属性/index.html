依赖属性 (Dependency Property) vs 附加属性 (Attached Property)

1. 核心概念

  • 依赖属性 (DP)：

    • 定义：由 DependencyObject 类提供的属性系统。它不直接存储在对象的字段中，而是存储在 WPF 的属性系统中。
    • 所有权：属性属于定义它的类。
    • 目的：提供样式、动画、数据绑定、属性值继承、默认值等高级功能。
    • 例子：Button.Width, TextBox.Text, Control.Background。

  • 附加属性 (AP)：

    • 定义：是一种特殊的依赖属性。它允许在一个对象上设置一个属性，而这个属性实际上是由另一个类定义的。
    • 所有权：属性由类 A 定义，但设置在类 B 的实例上。
    • 目的：在不修改类源代码的情况下，给对象附加额外的信息或行为。
    • 例子：Grid.Row, Grid.Column, DockPanel.Dock。

2. 核心区别对比表

特性 依赖属性 (DP) 附加属性 (AP)
注册方法 DependencyProperty.Register DependencyProperty.RegisterAttached
访问方式 (C#) 通过 CLR 包装器 (get/set) 通过静态 GetXXX / SetXXX 方法
XAML 用法 <Element Property="Value" /> <Element Owner.Property="Value" />
存储位置 对象实例的属性存储区 对象实例的属性存储区 (但由所有者管理)
典型场景 自定义控件的属性、样式化 布局定位、附加行为、MVVM 消息传递

3. 代码实现示例

依赖属性 (DP) 实现

通常用于自定义控件，暴露可绑定的属性。

public class MyControl : Control
{
    // 1. 注册
    public static readonly DependencyProperty IsActiveProperty =
        DependencyProperty.Register("IsActive", typeof(bool), typeof(MyControl), 
            new PropertyMetadata(false, OnIsActiveChanged));

    // 2. CLR 包装器 (方便 C# 代码访问)
    public bool IsActive
    {
        get { return (bool)GetValue(IsActiveProperty); }
        set { SetValue(IsActiveProperty, value); }
    }

    // 3. 属性变化回调
    private static void OnIsActiveChanged(DependencyObject d, DependencyPropertyChangedEventArgs e)
    {
        // 处理逻辑
    }
}

XAML 使用：

<local:MyControl IsActive="True" />

附加属性 (AP) 实现

通常用于工具类，给现有控件"打标签"。

public static class MouseBehavior
{
    // 1. 注册 (注意 RegisterAttached)
    public static readonly DependencyProperty IsClickEnabledProperty =
        DependencyProperty.RegisterAttached("IsClickEnabled", typeof(bool), typeof(MouseBehavior), 
            new PropertyMetadata(false, OnIsClickEnabledChanged));

    // 2. 静态 Get/Set 方法 (必须)
    public static bool GetIsClickEnabled(UIElement element)
    {
        return (bool)element.GetValue(IsClickEnabledProperty);
    }

    public static void SetIsClickEnabled(UIElement element, bool value)
    {
        element.SetValue(IsClickEnabledProperty, value);
    }

    // 3. 回调逻辑 (例如在这里附加事件监听)
    private static void OnIsClickEnabledChanged(DependencyObject d, DependencyPropertyChangedEventArgs e)
    {
        if (d is UIElement element)
        {
            if ((bool)e.NewValue)
                element.MouseLeftButtonUp += Element_MouseLeftButtonUp;
            else
                element.MouseLeftButtonUp -= Element_MouseLeftButtonUp;
        }
    }
}

XAML 使用：

<!-- 给 Button 附加了一个它本身没有的属性 -->
<Button Content="Click Me" local:MouseBehavior.IsClickEnabled="True" />

4. 使用场景总结

  • 依赖属性：当你开发自定义控件，需要支持数据绑定、动画或样式时，必须使用 DP。
  • 附加属性：
    1. 布局：如 Grid.Row，子控件不需要知道 Grid 的存在，但 Grid 需要知道子控件的行号。
    2. 附加行为：如上面的示例，给普通控件附加点击逻辑、拖拽功能等。
    3. MVVM 扩展：在不修改控件代码的情况下，将事件绑定到 ViewModel 命令（如 EventToCommand 本质就是利用附加属性）。

关系说明

  • Ghostty：终端窗口本身，是你看到的那个「应用程序」
  • Fish：你在终端里实际使用的 Shell，也就是命令交互层
  • Starship：显示在命令行前面的提示行（Prompt）引擎
  • Catppuccin：一套视觉主题设计语言，负责整体颜色风格

这四者的关系不是平行竞争，而是上下堆叠、彼此配合。它们真正的逻辑顺序应该这样理解：

Ghostty（终端窗口）
└── Fish（Shell / 命令交互层）
    └── Starship（Prompt 显示层）
        └── Catppuccin（视觉风格 / 配色系统）

Ghostty 是容器，Fish 是操作核心，Starship 是信息面板，Catppuccin 是美术系统。

Ghostty

Ghostty 是一个现代终端模拟器（terminal emulator）。它负责提供你可以看见、输入、输出命令的窗口环境。

  • 开启终端窗口
  • 显示文字
  • 管理字体与字型渲染
  • 应用主题背景颜色
  • 处理光标、分页、窗口行为

Ghostty 的定位很像是「工作桌面」：你要先有桌子，后面才能摆上各种工具。

Fish

Fish 是一个 Shell。当你在终端中输入命令，真正负责理解这些命令、执行它们、把结果回传给你的，不是 Ghostty，而是 Shell。

Fish 相较于 bash 或 zsh，最大的特色是：

  • 默认就有很好的自动补全
  • 会根据历史命令给出建议
  • 语法更直观
  • 日常交互体验很流畅

你可以把 Fish 想成是「房间里真正和你对话的工作伙伴」。Ghostty 只是把房间打开，而 Fish 才是你进到房间后，真正开始交互的对象。

Starship

Starship 是一个 Prompt 引擎。所谓 Prompt，就是你在终端里每次输入命令之前，前面看到的那一行提示信息。

~/project  main ❯

Starship 不取代 Shell，也不取代终端模拟器。它只是接管 Prompt 显示层，让你的提示行显示：

  • 当前所在文件夹
  • 当前 git branch 与状态
  • 上一个命令执行时间
  • 当前语言环境（Python、Node.js、Rust 等）

所以 Starship 很像是「驾驶座前方的仪表盘」。Fish 负责让车子真的能开，Starship 则负责把速度、油量、方向等信息显示在你面前。

Catppuccin

Catppuccin 不是终端、不是 Shell、也不是 Prompt 引擎。它本质上是一套 主题配色系统，而且是跨工具、跨环境、风格一致的色彩设计。

  • 终端主题
  • 编辑器主题
  • Prompt 配色
  • 各种开发工具的色彩风格

特别是 Mocha 版本，走的是柔和、低刺激、深色基底的路线，非常适合长时间使用。Catppuccin 更像是这整套系统的「视觉语言」。它本身不处理逻辑，但它决定了整体气氛与感受。

它们之间的实际工作流程是什么？

1. 打开 Ghostty

它负责提供终端窗口环境。

2. Ghostty 加载 Fish Shell

Fish 接手命令交互层，读取自己的配置文件。

3. Fish 初始化 Starship

Starship 接管 Prompt 显示，把当前目录、git branch、命令耗时整理成提示行。

4. Catppuccin 同时作用在两边

Ghostty 背景与 Starship 区块颜色都采用 Catppuccin，形成视觉一致的整体风格。

搞清楚层级分工，问题就变得很容易排查：

  • Powerline 图示乱码 → 先检查 Ghostty 的字体设置
  • 命令补全有问题 → 看 Fish 的设置
  • git branch 不显示 → 检查 Starship 设置
  • 整体颜色不一致 → 回头看 Catppuccin 是否两边都有应用

实际配置

先装 Ghostty

如果使用 macOS，可以先用 Homebrew 安装 Ghostty：

brew install --cask ghostty

再来安装 Fish 

#安装fish
brew install fish

#切换到fish
fish

#添加homebrew路径，否则brew命令无法在fish终端上运行
fish_add_path /opt/homebrew/bin

#获取fish的路径，会输出 /opt/homebrew/bin/fish
which fish

#添加 Fish 路径到 /etc/shells
sudo sh -c 'echo /opt/homebrew/bin/fish >> /etc/shells'

#改变默认shell
chsh -s /opt/homebrew/bin/fish

#验证
echo $SHELL

接着安装 Starship 

brew install starship

在 ~/.config/fish/config.fish 末尾加入：

starship init fish | source

最后让 Catppuccin 接管整体视觉

Ghostty 配置中加入：

theme = Catppuccin Mocha

这样你的终端背景、提示行、区块过渡色，就会形成一套完整的视觉系统。如果在配置自己的终端感觉麻烦也可以参考 lewislulu 网友的仓库，可以直接安装配置。

参考自: https://www.samingblog.com/ghostty-fish-starship-catppuccin-guide/

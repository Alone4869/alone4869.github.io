Behavior (行为) vs Trigger (触发器)

这两者都是为了让 XAML 具备交互逻辑，减少 Code-Behind 代码，但它们的层级和能力不同。

1. Trigger (触发器)

  • 定义：WPF 原生核心功能 (System.Windows)。
  • 机制：监听某个条件的变化（属性值、数据值、事件），当条件满足时，执行一组 Setter（设置属性）或 Animation（动画）。
  • 分类：
    1. PropertyTrigger：监听依赖属性变化（如 IsMouseOver）。
    2. DataTrigger：监听数据绑定值变化（如 ViewModel.Status == "Error"）。
    3. EventTrigger：监听路由事件（注意：原生 WPF 的 EventTrigger 只能执行动画 Storyboard，不能直接调用命令）。
  • 位置：通常定义在 Style.Triggers 或 ControlTemplate.Triggers 中。

Trigger 示例 (改变背景) 

<Style TargetType="Button">
    <Style.Triggers>
        <!-- 当鼠标悬停时，改变背景 -->
        <Trigger Property="IsMouseOver" Value="True">
            <Setter Property="Background" Value="LightBlue"/>
        </Trigger>
        <!-- 当数据为真时，隐藏控件 -->
        <DataTrigger Binding="{Binding IsVisible}" Value="False">
            <Setter Property="Visibility" Value="Collapsed"/>
        </DataTrigger>
    </Style.Triggers>
</Style>

2. Behavior (行为)

  • 定义：WPF 扩展库功能 (旧版 System.Windows.Interactivity, 新版 Microsoft.Xaml.Behaviors.Wpf)。
  • 机制：一个可复用的组件，附加到 UI 元素上。它可以监听事件、执行命令、调用方法、操作 DOM 树等。
  • 能力：比 Trigger 强大得多。它可以封装复杂的 C# 逻辑，并且完美支持 MVVM 命令绑定。
  • 位置：定义在 Interaction.Behaviors 集合中。
  • 依赖：需要安装 NuGet 包 Microsoft.Xaml.Behaviors.Wpf。

Behavior 示例 (点击命令绑定)

原生 Button 有 Command 属性，但 TextBox 没有。Behavior 可以让 TextBox 支持回车触发命令。

<!-- 需引入命名空间 xmlns:i="http://schemas.microsoft.com/xaml/behaviors" -->
<TextBox>
    <i:Interaction.Behaviors>
        <!-- 当 KeyDown 事件发生，且参数是 Enter 键时，执行 SaveCommand -->
        <i:EventTrigger EventName="KeyDown">
            <i:InvokeCommandAction Command="{Binding SaveCommand}" 
                                   CommandParameter="{Binding RelativeSource={RelativeSource AncestorType=TextBox}, Path=Text}"/>
        </i:EventTrigger>
    </i:Interaction.Behaviors>
</TextBox>

3. 核心区别对比表

特性 Trigger (触发器) Behavior (行为)
来源 WPF 核心库 (System.Windows) 扩展库 (Microsoft.Xaml.Behaviors)
主要用途 UI 状态变化 (视觉反馈) 交互逻辑 (事件处理、命令调用)
执行动作 Setter (改属性), Storyboard (动画) Command, Method, Action
MVVM 支持 较弱 (EventTrigger 原生不支持 Command) 极强 (内置 InvokeCommandAction)
复用性 通常写在 Style 里复用 本身就是独立的类，极易复用
代码侵入 纯 XAML 可能需要写少量 C# 类继承 Behavior
性能 轻量，原生优化 略重，但可接受

4. 详细使用场景与选择指南

场景 1：鼠标悬停变色、数据错误变红

  • 选择：Trigger
  • 理由：这是纯粹的 UI 状态映射，不需要执行逻辑代码，Trigger 性能最好且无需额外库。 
    <DataTrigger Binding="{Binding HasError}" Value="True">
    <Setter Property="BorderBrush" Value="Red"/>
</DataTrigger>

场景 2：ListBox 双击项执行命令

  • 选择：Behavior
  • 理由：ListBox 没有 DoubleClickCommand 属性。原生 EventTrigger 只能播动画。必须用 Behavior 的 EventToCommand 功能将事件转为命令。 
    <ListBox ItemsSource="{Binding Items}">
    <i:Interaction.Triggers>
        <i:EventTrigger EventName="MouseDoubleClick">
            <i:InvokeCommandAction Command="{Binding ItemDoubleClickCommand}" 
                                   CommandParameter="{Binding SelectedItem}"/>
        </i:EventTrigger>
    </i:Interaction.Triggers>
</ListBox>

场景 3：窗口加载时自动聚焦某个 TextBox

  • 选择：Behavior
  • 理由：Loaded 事件触发后需要调用 Focus() 方法。Trigger 无法调用方法。可以写一个 FocusOnLoadBehavior。 
    <TextBox>
    <i:Interaction.Behaviors>
        <local:FocusOnLoadBehavior />
    </i:Interaction.Behaviors>
</TextBox>

场景 4：拖拽文件到窗口

  • 选择：Behavior (或附加属性)
  • 理由：涉及复杂的 DragEnter, DragDrop 事件处理和逻辑判断。封装成 Behavior 可以在任何控件上复用拖拽功能。

5. 自定义 Behavior 示例

如果你想封装复杂逻辑，可以继承 Behavior<T>。

// 1. 定义 Behavior
public class TextBoxNumericBehavior : Behavior<TextBox>
{
    protected override void OnAttached()
    {
        base.OnAttached();
        this.AssociatedObject.TextChanging += OnTextChanging;
    }

    protected override void OnDetaching()
    {
        this.AssociatedObject.TextChanging -= OnTextChanging;
        base.OnDetaching();
    }

    private void OnTextChanging(object sender, TextChangingEventArgs e)
    {
        // 2. 在这里写 C# 逻辑，例如只允许输入数字
        var textBox = sender as TextBox;
        // ... 过滤逻辑 ...
    }
}

XAML 使用：

<TextBox>
    <i:Interaction.Behaviors>
        <local:TextBoxNumericBehavior />
    </i:Interaction.Behaviors>
</TextBox>

6. 总结与建议

  1. 优先使用 Trigger：如果只是改变 UI 外观（颜色、可见性、大小），Trigger 是原生、轻量且标准的选择。
  2. 必须使用 Behavior 的情况：
    • 需要将非命令事件（如 MouseDoubleClick, KeyDown, Loaded）绑定到 ViewModel 的 ICommand。
    • 需要复用复杂的交互逻辑（如拖拽、水印、自动聚焦）。
    • Trigger 无法实现的逻辑（调用方法、操作集合）。
  3. 关于 EventTrigger 的误区：
    • WPF 原生的 <EventTrigger> 只能包含 <BeginStoryboard>。
    • 如果你看到 XAML 里 <EventTrigger> 里面用了 <InvokeCommandAction>，那其实已经隐式依赖了 Microsoft.Xaml.Behaviors 库，这属于 Behavior 的范畴（尽管写法像 Trigger，在 Behaviors 库中它被归类为 Trigger 的扩展）。
  4. 库的选择：
    • 老项目可能引用 System.Windows.Interactivity.dll。
    • 新项目请务必使用 NuGet 包 Microsoft.Xaml.Behaviors.Wpf，这是微软官方维护的开源版本，支持 .NET Core 和 .NET 5/6/7/8。

通过合理组合 依赖属性/附加属性（数据通道）与 Trigger/Behavior（交互逻辑），可以实现几乎零 Code-Behind 的纯 MVVM WPF 应用。
